Tutorial: El Nodo de Pinata

Pinata es un "Pinning Service". IPFS funciona por popularidad: si nadie accede a tu archivo, los nodos lo borran (Garbage Collection).

Al "pinear" un archivo en Pinata, te aseguras de que sus nodos siempre mantengan una copia activa de tu NFT, garantizando que nunca desaparezca.

¿Cómo usar la API?
// Ejemplo rápido con JavaScript
const pinata = new PinataSDK({ 
  pinataJWTKey: 'TU_JWT_AQUI' 
});

const res = await pinata.pinFileToIPFS(archivo);
console.log("Tu CID es:", res.IpfsHash);

Diferencias de Código: ERC-721 vs ERC-1155

ERC-721 (NFT Único)

Cada ID es único. Ideal para obras de arte, dominios (.eth) o ítems coleccionables raros.

// Cada mint crea un ID nuevo
function mintUnique(address to) public {
    _tokenIds++;
    _safeMint(to, _tokenIds);
}

ERC-1155 (Multi-Token)

Un mismo ID puede tener múltiples dueños. Ideal para items de juegos (pociones, munición).

// Un ID con 1000 copias
function mintBatch(address to) public {
    _mint(to, POCION_ID, 1000, "");
}

IPFS: ¿Por qué es vital?

El Problema de la Web2:

Si guardas tu foto en `tuweb.com/foto.jpg` y dejas de pagar el hosting, el NFT queda "roto" (enlace muerto). El dueño del NFT pierde su arte.

La Solución Web3:

IPFS usa **Content Addressing**. El link es el ADN del archivo. Mientras alguien en el mundo tenga una copia, tu arte seguirá existiendo.
